= gitlab-code-quality-plugin
:version_stable: 1.1.0
:version_snapshot: 1.2.0-SNAPSHOT

This Gradle plugin transforms XML reports produced by tools such as
https://spotbugs.github.io/[SpotBugs] and https://checkstyle.org/[Checkstyle]
into JSON in the format expected by https://about.gitlab.com/[GitLab], so that
findings can be shown in the merge request code quality widget.

image::.readme/gitlab-merge-request-widget.png[]

== Usage

=== Step 1: Configure SpotBugs and/or Checkstyle

The plugin only reads XML produced by other tools. Configure the SpotBugs and/or Checkstyle Gradle plugins (or any setup that writes compatible XML) so reports exist before `+check+` runs.

Default input locations are `+build/spotbugsXml.xml+` and `+build/checkstyle-result.xml+`. Point your SpotBugs/Checkstyle tasks at those paths, or override the `+generateGitLabCodeQualityReport+` task properties (see <<advanced-configuration>>).

=== Step 2: Apply this plugin

The `+java+` plugin must be applied on the same project (directly or transitively, for example via Kotlin/JVM). Then add this plugin to your `+build.gradle.kts+` (or `+build.gradle+`):

[source,kotlin,subs="+attributes"]
----
plugins {
    java
    id("io.github.dnalchemist.gitlab-code-quality") version "{version_stable}"
}
----

By default the plugin reads:

* `+build/spotbugsXml.xml+`
* `+build/checkstyle-result.xml+`

and writes:

* `+build/gl-code-quality-report.json+`

when the `+check+` lifecycle runs (the `+generateGitLabCodeQualityReport+` task is attached to `+check+`).

If the XML files are missing or empty, the task still completes and logs that no report was found.

=== Step 3: Publish the report to GitLab CI

Point GitLab at the generated JSON, for example:

[source,yaml]
----
build:
  stage: build
  image: ...
  script:
    - ./gradlew check
  artifacts:
    reports:
      codequality:
        - build/gl-code-quality-report.json
----

[[advanced-configuration]]
== Advanced configuration

All options are properties on the `+generateGitLabCodeQualityReport+` task. Example:

[source,kotlin,subs="+attributes"]
----
import io.github.dnalchemist.gitlab.codequality.GenerateGitLabCodeQualityReportTask

tasks.named<GenerateGitLabCodeQualityReportTask>("generateGitLabCodeQualityReport") {
    spotbugsEnabled.set(true)
    spotbugsInputFile.set(layout.buildDirectory.file("spotbugsXml.xml"))
    checkstyleEnabled.set(true)
    checkstyleInputFile.set(layout.buildDirectory.file("checkstyle-result.xml"))
    outputFile.set(layout.buildDirectory.file("gl-code-quality-report.json"))
}
----

== Multi-project builds

Apply the plugin in subprojects that run SpotBugs/Checkstyle. Each project produces its own JSON file under its `+build/+` directory.

GitLab accepts one code quality file per job (see https://gitlab.com/gitlab-org/gitlab/-/issues/9014[this issue]). Merge reports with https://stedolan.github.io/jq/[jq], for example:

[source,yaml]
----
build:
  stage: build
  image: ...
  before_script:
    - apt-get update && apt-get install -y jq
  script:
    - ./gradlew check
  after_script:
    - find . -name gl-code-quality-report.json -print0 | xargs -0 cat | jq -s "add" > merged-gl-code-quality-report.json
  artifacts:
    reports:
      codequality:
        - merged-gl-code-quality-report.json
----

== Using snapshots

Snapshots are published to the Sonatype OSSRH snapshots repository. Add that repository for *plugin* resolution in `+settings.gradle.kts+`, then apply the snapshot version in your build script:

[source,kotlin,subs="+attributes"]
----
// settings.gradle.kts
pluginManagement {
    repositories {
        gradlePluginPortal()
        maven { url = uri("https://s01.oss.sonatype.org/content/repositories/snapshots/") }
    }
}
----

[source,kotlin,subs="+attributes"]
----
// build.gradle.kts
plugins {
    java
    id("io.github.dnalchemist.gitlab-code-quality") version "{version_snapshot}"
}
----
